# 组件原语

> 原语先于业务组件。Button、Card、Input、Badge、Progress 等基础件必须先定义状态，再被页面复用。

---

## 通用规则

- Props 必须强类型化，禁止 `any`。
- 组件只使用设计 token，不写临时颜色、间距、圆角、阴影。
- 所有可交互组件必须覆盖 default、hover、active、focus-visible、disabled。
- 数据组件必须覆盖 loading、empty、error。
- 图标极度克制：优先使用几何符号或 lucide 单色线性图标，不使用廉价彩色图标和 emoji。

---

## Button

| Variant | 用途 | 视觉 |
| --- | --- | --- |
| `primary` | 页面主行动 | `--gradient-coral`、pill、柔和阴影 |
| `secondary` | 次行动 | 浅纸面、深鸭蓝文字 |
| `ghost` | 低权重行动 | 无填充，只保留 hover wash |
| `icon` | 工具按钮 | 正圆或正方圆角，必须有 aria-label |

```ts
export type ButtonVariant = 'primary' | 'secondary' | 'ghost' | 'icon';

export interface ButtonProps {
  variant?: ButtonVariant;
  loading?: boolean;
  disabled?: boolean;
  children: React.ReactNode;
}
```

按钮圆角必须是 `--radius-full`，触控高度不小于 `44px`。

---

## Card

| Variant | 用途 |
| --- | --- |
| `surface` | 默认内容卡 |
| `raised` | 重点卡、悬浮卡 |
| `dark` | 对比面板、进度状态 |
| `inset` | 内嵌选择区、表单组 |

Card 不嵌套 Card。页面区块不要伪装成一张大卡，只有重复项目、弹窗和真实工具面板使用 Card。

---

## Input / Textarea

必须包含：

- label
- placeholder
- helper text
- error message
- focus-visible
- disabled

```ts
export interface FieldState {
  status?: 'default' | 'success' | 'error';
  message?: string;
}
```

输入框使用 `--color-surface-raised` 或 `--color-surface-inset`，错误状态用 `--color-error` 和 `--color-error-bg`，不使用刺眼纯色。

---

## Badge / Status

默认使用 whisper style：小圆点 + 文字，不使用厚重胶囊。

```tsx
<span className="status-badge">
  <span className="status-dot" aria-hidden="true" />
  <span>RUNNING</span>
</span>
```

需要强调时才使用 pill badge，并且只使用 `--radius-full`。

---

## Avatar

- 有真实头像时使用真实图片。
- 无头像时使用姓名首字母 fallback。
- 颜色从点缀色 token 中选择，不随机生成。
- 图片必须有 alt 或被明确标记为装饰。

---

## Progress

进度条使用 `Pill-in-Pill` 模式：

- track 厚度不小于 `--space-24`。
- track 内部有 `--space-4` padding。
- fill 是实心 pill，不贴边。
- 深色面板内状态色使用 `08-dark-panel-tokens.md`。

---

## Modal / Drawer

- 支持 ESC 关闭。
- 支持点击遮罩关闭，危险操作除外。
- 打开时锁定背景滚动。
- 初始焦点落在标题或第一个可操作控件。
- 关闭后焦点回到触发器。

---

## Toast

Toast 只用于短反馈，不承载长解释。

| 状态 | Token |
| --- | --- |
| success | `--color-success` |
| warning | `--color-warning` |
| error | `--color-error` |
| info | `--color-info` |

---

## 空、错、加载

| 状态 | 要求 |
| --- | --- |
| Loading | skeleton 或明确 loading copy |
| Empty | 告诉用户当前没有内容，并给出下一步 |
| Error | 说明失败原因，提供重试或返回路径 |
| Disabled | 显示不可用原因，不只降低透明度 |
