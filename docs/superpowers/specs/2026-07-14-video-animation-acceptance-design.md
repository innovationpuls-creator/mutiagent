# 视频检索与 HTML 动画验收修复设计

## 现状证据

2026-07-14 在腾讯云生产环境重新触发章节生成后，backend 日志确认：

- `POST /api/chat/start` 返回 200，耗时 7.039 秒。
- `POST /api/chat/message` 返回 200，耗时 3.642 秒。
- Markdown 小节智能体完成后进入视频检索阶段。
- 视频检索阶段连续记录 `YouTube search failed for query ...`，覆盖当前章节的多个查询。
- Bilibili 搜索没有记录异常，但当前实现只在响应正文中匹配 BV 号；没有 BV 号时返回空列表。
- 现有视频质量门随后把空结果归类为 `视频生成失败`。
- 生产页面的动画 iframe 中实际出现 `&oklch(72% 0.08 240);` 文本节点，说明非法颜色文本穿过了 HTML 清洗和质量检查。

因此，本次修复不改变“必须是真实可打开视频页面”的验收要求，也不把搜索页兜底标记为成功。

## 目标

1. 视频检索失败时能够区分平台请求失败、平台返回无可解析结果、视频元数据校验失败和主题质量不合格。
2. 对当前腾讯云出口下的 YouTube 失败和 Bilibili 无 BV 结果提供可诊断日志。
3. 只有真实 HTTP(S) 视频页面通过 brief 绑定、URL、平台元数据和主题质量校验时，视频资源才标记为 `available`。
4. HTML 动画不得把非法 CSS 文本节点、代码围栏、编码残片或不可见结构写入 iframe。
5. 动画质量检查必须覆盖实际 HTML 结构：实体、关系、时间线、中文上下文、颜色契约和可见性降级。
6. 动画失败必须保留明确失败状态，不把低质量 HTML 当作成功。

## 方案

### 视频检索边界

保留 Bilibili 与 YouTube 并行检索和章节 45 秒总超时。每个平台搜索包装器记录：

- 平台名称
- 小节 ID
- 查询文本
- 请求耗时
- HTTP 状态或异常类型
- 原始结果数量
- 解析后结果数量

YouTube 搜索失败时记录异常类型和异常信息；解析不到 `ytInitialData` 或 `videoRenderer` 时分别记录响应解析阶段。Bilibili 搜索同样记录响应状态、响应正文是否包含 BV 号和解析结果数量。

搜索阶段不引入搜索页或人工拼接 URL 作为成功结果。平台无结果继续执行质量门，最终返回结构化的 `unavailable` 结果和准确失败原因。

### HTML 动画输入清洗

在 `_normalize_animation_html` 之后、质量检查之前增加严格的 HTML 规范化边界：

- 去除 HTML/Markdown 代码围栏。
- 将 `&oklch(...)`、`&rgb(...)`、`&hsl(...)` 这类编码残片识别为非法 CSS 文本并拒绝。
- 只允许 OKLCH 或 CSS 变量颜色表达。
- 要求存在一个 `section-animation` 根节点，且实体、关系和时间线元素位于该根节点内。
- 保留 `prefers-reduced-motion` 的 `opacity` 和 `transform` 降级规则。

如果模型输出不满足契约，使用现有确定性动画生成器依据同一份 `animation_briefs` 重建结构动画；重建结果仍必须经过同一质量门。重建不是把失败标记为成功，而是只有确定性结果完整通过检查时才返回 `available`。

### 前端可见性验收

不改变 iframe 的安全沙箱边界。后端在持久化前完成结构性检查，前端继续使用 `srcDoc` 渲染；新增测试覆盖 iframe 输入中非法 CSS 残片不会作为可见文本保留。页面上的失败卡片继续显示失败状态，不展示损坏 HTML。

## 错误处理

- 平台网络错误：记录异常类型、异常文本和平台，继续另一平台检索。
- 平台响应无可解析结果：记录阶段和结果数量，继续质量门。
- 45 秒章节超时：返回 `视频检索超时`，并保留平台耗时日志。
- 动画非法 HTML：记录具体质量问题，进入确定性重建；重建失败则返回动画失败。
- 任何真实资源未通过校验：不写入 `available`，不把 fallback 搜索页当成成功。

## 测试与验收

新增或补充以下测试：

1. YouTube 请求异常记录异常类型，Bilibili 无 BV 结果记录解析结果数量。
2. 两个平台都无结果时视频状态为 `unavailable`，失败原因准确且视频数组为空。
3. 真实视频通过 URL、brief 绑定和元数据主题校验后才为 `available`。
4. `&oklch(...)`、`&rgb(...)` 等残片不能通过动画 HTML 质量门。
5. 动画实体、关系、时间线和 `prefers-reduced-motion` 缺失时不能标记成功。
6. 确定性动画重建结果包含真实可见实体、关系线和步骤交互，并通过现有质量检查。
7. 运行后端定向契约测试、完整后端测试和 Ruff；部署后重新生成 1.1，核对 backend 日志与页面实际结果。

## 不在范围内

- 不引入第三方视频 API 或新的代理服务。
- 不把搜索页、推荐页或无法验证的 URL 作为成功视频。
- 不重写课程 Markdown 生成链。
- 不做与本次视频、动画验收无关的前端视觉重构。
