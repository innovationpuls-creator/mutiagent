# 一棵树 (OneTree) 数据库表结构

一棵树系统的后端采用 SQLite（测试/本地环境）和 PostgreSQL（生产环境）作为存储介质，数据表通过 **SQLModel**（结合 Pydantic 与 SQLAlchemy）进行声明式定义。

## 技术说明
* **ORM 框架**：SQLModel
* **JSON 支持**：生产环境中使用 PostgreSQL `JSONB` 格式存储，本地/测试环境中使用 SQLite 并在底层自动序列化为 TEXT 格式。
* **时区规范**：所有 `created_at`、`updated_at`、`published_at` 等时间字段，均使用 UTC 时间。

## 1. user（用户表）

存储系统内所有的学生、教师及管理员账号信息。

| 字段 | 类型 | 约束 | 说明 |
| :--- | :--- | :--- | :--- |
| `uid` | str | PK | 用户唯一标识 (UUID 字符串) |
| `username` | str | NOT NULL, 1-64 字符, 索引 | 用户名/显示名 |
| `identifier` | str | NOT NULL, 3-128 字符, 唯一索引 | 登录标识（手机号、邮箱或第三方登录 ID） |
| `role` | str | 默认 "student", 索引, 最大长 16 | 角色：`student` / `teacher` / `admin` |
| `school` | str | 默认 "", 索引, 最大长 128 | 学校名称 |
| `major` | str | 默认 "", 索引, 最大长 128 | 所属专业 |
| `class_name` | str | 默认 "", 索引, 最大长 128 | 班级名称 |
| `provider` | str | 默认 "password", 索引, 最大长 32 | 认证方式：`password` / `qq` / `xuexitong` |
| `password_hash`| str | 可空, 最大长 128 | 密码哈希（provider 为 password 时必填） |
| `is_active` | bool | 默认 True | 账户是否处于激活状态 |
| `created_at` | datetime| UTC, 默认当前时间 | 账号注册创建时间 |
| `updated_at` | datetime| UTC, 默认当前时间 | 信息更新时间 |
| `last_login_at`| datetime| 可空 | 最后一次成功登录的时间 |

## 2. cultivationprogram（人培方案表）

由教师或管理员发布，供某个特定班级/专业匹配的通用学习大纲模板。

| 字段 | 类型 | 约束 | 说明 |
| :--- | :--- | :--- | :--- |
| `program_id` | str | PK, 最大长 64 | 培养方案唯一标识 |
| `teacher_uid` | str | FK ➡️ `user.uid`, 索引 | 发布该培养方案的教师用户 ID |
| `school` | str | NOT NULL, 联合唯一, 索引, 最大 128 | 关联的学校 |
| `major` | str | NOT NULL, 联合唯一, 索引, 最大 128 | 关联的专业 |
| `class_name` | str | NOT NULL, 联合唯一, 索引, 最大 128 | 关联的班级 |
| `courses` | JSON | 默认 [] | 包含的课程大纲节点列表 |
| `published_at` | datetime| 可空 | 方案发布时间 (为 null 表示草稿状态) |
| `created_at` | datetime| UTC | 创建时间 |
| `updated_at` | datetime| UTC | 更新时间 |

* **唯一约束**：`school + major + class_name` 建立联合唯一约束。每个班级只能拥有一套有效的培养方案。

## 3. userprofile（用户画像表）

与 `user` 表 1:1 关联，存储多智能体收集并生成的结构化学生特征和自然语言总结。

| 字段 | 类型 | 约束 | 说明 |
| :--- | :--- | :--- | :--- |
| `user_uid` | str | PK, FK ➡️ `user.uid` | 关联的用户 ID |
| `profile_data` | JSON | 默认 {} | 画像结构化数据（含目标、强弱项、时间分配等） |
| `profile_text` | str | 默认 "" | 由画像智能体生成的对该学生的自然语言评估全文 |
| `created_at` | datetime| UTC | 创建时间 |
| `updated_at` | datetime| UTC | 更新时间 |

---

## 4. useryearlearningpath（用户学年路径表）

根据用户画像，由路径规划智能体生成的按学年定制的课程节点路径图。

| 字段 | 类型 | 约束 | 说明 |
| :--- | :--- | :--- | :--- |
| `user_uid` | str | PK, FK ➡️ `user.uid`, 联合索引 | 关联的用户 ID |
| `grade_year` | str | PK, 最大长 16 | 学年标识：`year_1` / `year_2` / `year_3` / `year_4` |
| `learning_topic`| str | 默认 "", 最大长 256 | 该学年的核心学习主题/方向 |
| `path_data` | JSON | 默认 {} | 该学年的详细课程节点拓扑关系及当前学习节点状态 |
| `created_at` | datetime| UTC | 创建时间 |
| `updated_at` | datetime| UTC, 联合索引 | 更新时间 |

* **联合索引**：在 `(user_uid, updated_at)` 上建立索引，便于快速检索最近的路径更新记录。

---

## 5. usercourseknowledgeoutline（课程知识大纲表）

精读展叶阶段的核心数据，存储单门课程被拆分出的详细章节与知识层级大纲。

| 字段 | 类型 | 约束 | 说明 |
| :--- | :--- | :--- | :--- |
| `user_uid` | str | PK, FK ➡️ `user.uid`, 联合索引 | 关联的用户 ID |
| `course_id` | str | PK, 最大长 128 | 课程节点 ID (如 `data-structure`) |
| `grade_year` | str | 联合索引, 最大长 16 | 所属学年 (如 `year_1`) |
| `course_name` | str | 默认 "", 最大长 256 | 课程名称 |
| `outline_data` | JSON | 默认 {} | 大纲结构数据（章节列表、章节要点、视频/动画关联等） |
| `created_at` | datetime| UTC | 创建时间 |
| `updated_at` | datetime| UTC | 更新时间 |

* **联合索引**：在 `(user_uid, grade_year)` 上建立索引，支持按学年打包获取用户大纲信息。

---

## 6. chapterquiz（章节测验表）

自测森林阶段的数据，记录为特定章节动态生成的测验题集。

| 字段 | 类型 | 约束 | 说明 |
| :--- | :--- | :--- | :--- |
| `quiz_id` | str | PK, 最大长 64 | 测验唯一 ID |
| `user_uid` | str | FK ➡️ `user.uid`, 联合唯一, 索引 | 关联的用户 ID |
| `course_node_id`| str | 联合唯一, 索引, 最大长 128 | 关联的课程 ID |
| `chapter_id` | str | 联合唯一, 索引, 最大长 64 | 关联的章节 ID |
| `status` | str | 默认 "ready", 索引, 最大长 24 | 测验状态：`ready` / `generating` / `failed` |
| `questions` | JSON | 默认 [] | 生成的题目对象列表（包含题干、选项、正确答案、解析） |
| `source_outline_version` | str | 默认 "", 最大长 64 | 对应大纲的版本哈希，用于检测大纲更新 |
| `generation_error` | str | 默认 "" | 若生成失败，记录大模型或服务报错信息 |
| `created_at` | datetime| UTC | 创建时间 |
| `updated_at` | datetime| UTC | 更新时间 |

* **联合唯一约束**：`user_uid + course_node_id + chapter_id` 联合唯一。即一个用户对特定章节只保留一套测验题集（可重复生成覆盖）。

---

## 7. chapterquizattempt（章节测验尝试表）

记录学生每次提交试卷后的答题明细以及批改打分结果。

| 字段 | 类型 | 约束 | 说明 |
| :--- | :--- | :--- | :--- |
| `attempt_id` | str | PK, 最大长 64 | 尝试记录唯一 ID |
| `quiz_id` | str | FK ➡️ `chapterquiz.quiz_id`, 联合索引, 最大 64 | 关联的测验 ID |
| `user_uid` | str | FK ➡️ `user.uid`, 联合索引 | 关联的用户 ID |
| `answers` | JSON | 默认 {} | 学生提交的答题键值对 (如 `{"q1": "A"}`) |
| `score` | int | 默认 0 | 测验得分 (百分制) |
| `passed` | bool | 默认 False, 索引 | 是否通关（达到合格线） |
| `grading_result`| JSON | 默认 {} | 批改报告（包含每题对错、详细反馈、薄弱知识点分析） |
| `created_at` | datetime| UTC | 提交及批改时间 |

* **联合索引**：在 `(user_uid, quiz_id)` 上建有索引，方便查询某测验的所有作答记录。

---

## 8. chapterprogress (章节解锁进度表)

追踪学生在课程中各章节的学习状态与自测解锁进度。

| 字段 | 类型 | 约束 | 说明 |
| :--- | :--- | :--- | :--- |
| `user_uid` | str | PK, FK ➡️ `user.uid`, 联合索引 | 关联的用户 ID |
| `course_node_id`| str | PK, 联合索引, 最大长 128 | 关联的课程 ID |
| `chapter_id` | str | PK, 最大长 64 | 关联的章节 ID |
| `state` | str | 默认 "locked", 索引, 最大长 24 | 状态：`locked` / `unlocked` / `passed` |
| `best_score` | int | 默认 0 | 该章节测验的最佳历史得分 |
| `latest_attempt_id` | str | 可空, 最大长 64 | 最新一次尝试记录的 ID |
| `passed_at` | datetime| 可空 | 首次通关的时间 |
| `updated_at` | datetime| UTC | 进度更新时间 |

---

## 9. chapterweakness（章节薄弱知识点表）

基于测验尝试中的批改结果，由智能体归纳出的薄弱知识点，用于指导学生复习和系统针对性推荐。

| 字段 | 类型 | 约束 | 说明 |
| :--- | :--- | :--- | :--- |
| `weakness_id` | str | PK, 最大长 64 | 薄弱点记录唯一 ID |
| `user_uid` | str | FK ➡️ `user.uid`, 索引 | 关联的用户 ID |
| `course_node_id`| str | 最大长 128 | 关联的课程 ID |
| `chapter_id` | str | 最大长 64 | 关联的章节 ID |
| `knowledge_point_id` | str | 最大长 128 | 对应薄弱知识点的标准化 ID |
| `knowledge_point_name` | str | 默认 "", 最大长 256 | 薄弱知识点中文名称 |
| `severity` | int | 默认 1, 值在 [1, 3] 区间 | 严重程度：1 (轻微), 2 (中度), 3 (严重) |
| `consumed` | bool | 默认 False, 索引 | 该薄弱点是否在后续复习/答疑中被消费/消除 |
| `created_at` | datetime| UTC | 分析产生时间 |

---

## 10. courseresourcequality（课程资源质量评估表）

记录大模型动态生成章节大纲、Markdown 文档以及交互式 HTML 动画等资源时的自动化质量评分，确保生成质量受控。

| 字段 | 类型 | 约束 | 说明 |
| :--- | :--- | :--- | :--- |
| `user_uid` | str | PK, FK ➡️ `user.uid`, 联合索引 | 关联的用户 ID |
| `course_node_id`| str | PK, 联合索引, 最大长 128 | 关联的课程 ID |
| `accuracy_score`| int | 默认 0, 值在 [0, 100] 区间 | 科学性/准确性评分 |
| `difficulty_fit_score` | int | 默认 0, 值在 [0, 100] 区间 | 难度匹配度评分 |
| `completeness_score` | int | 默认 0, 值在 [0, 100] 区间 | 内容完备性评分 |
| `overall_score` | int | 默认 0, 值在 [0, 100] 区间 | 综合质量评分 |
| `suggestions` | JSON | 默认 [] | 质量智能体给出的具体修改/优化建议列表 |
| `scored_at` | datetime| UTC | 评分执行时间 |
| `created_at` | datetime| UTC | 记录创建时间 |
| `updated_at` | datetime| UTC | 记录更新时间 |

---

## 11. conversationsession（会话消息表）

用于替代外部/内存 checkpointer 的持久化消息存储，归档用户与多智能体中枢的历史对话。

| 字段 | 类型 | 约束 | 说明 |
| :--- | :--- | :--- | :--- |
| `session_id` | str | PK | 会话 UUID 字符串 |
| `user_uid` | str | FK ➡️ `user.uid`, 联合索引 | 关联的用户 ID |
| `messages` | JSON | 默认 [] | 完整的 LangChain 会话历史消息明细列表 (Pydantic 序列化 JSON) |
| `created_at` | datetime| UTC | 创建时间 |
| `updated_at` | datetime| UTC, 联合索引 | 更新时间 |
