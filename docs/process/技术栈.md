# 前端技术栈

> 前端工程服务于 Headspace meditation 风格的中文用户界面：温暖、柔和、有呼吸感，同时保持 TypeScript 与设计 token 的严格约束。

---

## 核心框架

- React 18
- TypeScript strict mode
- Vite
- Node.js 包管理器按项目锁文件选择
- `react-router-dom` 用于页面级切换

所有组件 Props、hook 返回值、状态模型必须显式类型化，禁止 `any`。

---

## 样式架构

- Tailwind CSS 只用于布局与组合 class。
- 原生 CSS Variables 是设计 token 真相来源。
- 所有颜色来自 `docs/ui-design/01-颜色系统.md`。
- 所有字体来自 `docs/ui-design/02-字体系统.md`。
- 所有间距来自 `docs/ui-design/03-间距系统.md`。
- 所有圆角与阴影来自 `docs/ui-design/04-圆角与阴影.md`。
- 暗色、材质、动效、深色面板、组件原语分别按 `05-10` 文档执行。

禁止 Tailwind 任意颜色、任意 spacing、任意 shadow 替代 token。确实缺 token 时，先补文档和全局 CSS。

---

## 动效

- Framer Motion 负责复杂页面过渡和共享元素。
- CSS transition / animation 只处理轻量微交互。
- 动画只改变 `transform` 和 `opacity`。
- 默认使用 `type: "tween"`，按 `docs/ui-design/07-motion-physics.md` 映射 easing / duration。
- 必须支持 `prefers-reduced-motion`。

---

## 图标与素材

- 图标使用 lucide-react 或几何符号。
- 图标单色、低密度、克制使用。
- 缺图时使用明确 placeholder，不手画伪产品图。
- 不使用 emoji 作为图标替代。

---

## 开发流程

1. 先读设计系统文档。
2. 先声明 Design Decisions。
3. 先做 Layout Shell。
4. 再做 Hero 验证 token。
5. 再抽 Button / Card / Input 等 primitives。
6. 最后做动效、边缘状态和 polish。

具体流程与验收见 `docs/process/11-前端开发流程与验收.md`。
