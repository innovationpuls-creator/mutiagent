# 前端开发流程与验收

> 项目使用 Vibe Coding 顺序：先全局，后局部；先静态，后动态；先定义，再实现。

---

## 启动顺序

1. 调用 `/web-design-engineer` skill。
2. 阅读 `docs/ui-design/01-颜色系统.md` 到 `docs/process/11-前端开发流程与验收.md`。
3. 明确当前页面的叙事角色、观看距离、视觉温度、容量是否足够。
4. 先声明设计决策，再写代码。
5. 先做 viewable v0，确认方向后再补完整状态和动效。

---

## 8 阶段

| 阶段 | 产物 | 不做什么 |
| --- | --- | --- |
| 1. Design Tokens | 颜色、字体、间距、圆角、阴影、暗色、动效 | 不直接做 Hero |
| 2. Layout Shell | Header、Main、Footer、Container、Grid、断点 | 不填业务内容 |
| 3. Hero | 验证 token 同屏效果 | 不加复杂动画 |
| 4. Primitives | Button、Input、Card、Badge、Avatar、Progress | 不写一次性组件 |
| 5. Sections | 按页面流逐段实现 | 不跳着补局部 |
| 6. Motion | 入场、hover、路由、反馈 | 不动布局属性 |
| 7. Edge States | loading、empty、error、disabled、404、SEO | 不留默认白页 |
| 8. Polish | 真实文案、图片优化、a11y、性能、暗色微调 | 不用 Lorem Ipsum 糊弄 |

## v0 要求

v0 只解决方向，不追求完整：

- 有真实页面骨架。
- 有核心 token。
- 有 Hero 或关键模块占位。
- 明确列出设计假设。
- 用 `[image]`、`[icon]`、`[real data needed]` 标出缺失素材，不伪造数据。

---

## 实现要求

- React 18 + TypeScript。
- 组件 Props、状态、工具函数全部强类型化。
- CSS class 使用 kebab-case。
- 颜色、间距、圆角、阴影、动效必须来自 token。
- 不使用 `scrollIntoView`。
- 不使用 `const styles = { ... }` 作为全局样式对象名。
- 不添加无来源的统计数字、logo wall、testimonial。

---

## 前端验收

任何前端代码改动完成后必须验收：

1. 启动预览服务器或打开 HTML。
2. code-reviewer subagent 做代码审查。
3. e2e-runner subagent 使用 Playwright 验收。
4. 截图 `375px`、`768px`、`1280px`。
5. 检查 console 无 error / warning。
6. 验证 `prefers-reduced-motion`。
7. 检查无文字溢出、无组件重叠、无横向滚动。
8. 检查 hover、focus、active、disabled、loading、empty、error。

---

## 文档变更验收

只改文档时不需要启动前端预览，但必须检查：

- 必读文档是否齐全。
- 是否出现 HEX/RGB 硬编码示例。
- 是否出现任意 spacing / radius / shadow。
- 是否和 `AGENTS.md` 的硬规则冲突。
- 是否仍引用不存在的文档。
