# 颜色系统

> **核心哲学**：全面拥抱 Headspace 的「Warm Humanist」风格。禁用所有冷色调（如纯蓝、纯白、冷灰）。采用暖桃色作为画布底色，暖珊瑚色作为主要点缀，暖深鸭蓝色（Dark Teal）作为主要文字色（墨水色）。

---

## 品牌色 (Brand Colors)

完全基于 Headspace 的插画与品牌色彩规范：

| Token | 值 | 质感描述 | 用途 |
|-------|-----|---------|------|
| `--color-primary` | `oklch(76% 0.12 55)` `#F4A573` | **暖珊瑚 (Warm Coral)** | 品牌主色、主按钮、核心卡片背景、插画主要元素 |
| `--color-primary-hover` | `oklch(70% 0.12 55)` | | 主色 hover 状态（略微加深） |
| `--color-secondary` | `oklch(50% 0.04 240)` `#5C6B7A` | **柔和蓝灰 (Muted Blue-Grey)** | 辅助强调色、次要标签、图标 |
| `--color-accent-sage` | `oklch(74% 0.10 130)` `#9DB67A` | **鼠尾草绿 (Sage)** | 场景点缀色，交替使用 |
| `--color-accent-lavender` | `oklch(74% 0.07 290)` `#B0A5D1` | **淡紫 (Lavender)** | 场景点缀色，交替使用 |
| `--color-accent-salmon` | `oklch(70% 0.14 25)` `#F5867B` | **鲑鱼粉 (Salmon)** | 场景点缀色，交替使用 |

> **禁止事项**：禁止在同一个视口/卡片内同时出现 3 种以上的点缀色。每次场景应当以暖桃底色 + 深鸭蓝文字为主，辅以单一的点缀色。

---

## 背景 & 表面 (Ground & Surface)

告别纯白，引入极具放松感的底色：

| Token | 值 | 质感描述 | 用途 |
|-------|-----|---------|------|
| `--color-background` | `oklch(93% 0.05 70)` `#FFE2C5` | **暖桃色 (Warm Peach)** | 页面的全局基础底色 |
| `--color-surface` | `oklch(96% 0.03 70)` `#FFEDD5` | **浅桃色 (Light Peach)** | 基础卡片、浮层底色 |
| `--color-surface-pure` | `oklch(98% 0.01 70)` | **带暖意的白色** | 需要最高对比度时的纯净卡片 |
| `--color-border` | `oklch(85% 0.03 70)` | **柔和卡其** | 分割线、表单边框 |

---

## 文字与墨水 (Ink)

绝对禁止纯黑（`#000000`）。即使在最高对比度下，也必须使用带有色彩倾向的“墨水色”。

| Token | 值 | 质感描述 | 用途 |
|-------|-----|---------|------|
| `--color-text-primary` | `oklch(30% 0.05 240)` `#1B3A47` | **暖深鸭蓝 (Warm Dark Teal)** | 所有的正文、标题主要文字，取代传统的纯黑 |
| `--color-text-secondary` | `oklch(45% 0.04 240)` | **次级鸭蓝** | 副标题、补充说明文字 |
| `--color-text-muted` | `oklch(60% 0.02 240)` | **浅灰蓝** | 极淡的文字（如禁用态、输入框占位符） |
| `--color-text-inverse` | `oklch(98% 0.01 70)` | **反白文字** | 用于暖珊瑚色或点缀色背景上的文字 |

---

## 语义色 (Semantic Colors)

在 Headspace 风格中，传统的“大红大绿”警告色显得过于刺眼。所有的语义色都必须经过一定的灰度和明度中和：

| Token | 值 | 用途 |
|-------|-----|------|
| `--color-success` | `var(--color-accent-sage)` | 成功状态（复用鼠尾草绿，避免刺眼的纯绿） |
| `--color-success-bg` | `oklch(94% 0.04 130)` | 成功状态背景 |
| `--color-error` | `var(--color-accent-salmon)` | 错误状态（复用鲑鱼粉，避免血腥的纯红） |
| `--color-error-bg` | `oklch(94% 0.04 25)` | 错误状态背景 |
| `--color-info` | `var(--color-secondary)` | 信息提示（复用蓝灰） |
| `--color-info-bg` | `oklch(94% 0.02 240)` | 信息提示背景 |

---

## 交互态 (Interactive States)

| Token | 值 | 用途 |
|-------|-----|------|
| `--color-disabled` | `oklch(88% 0.01 70)` | 禁用态背景（不使用冷灰，使用暖灰） |
| `--color-disabled-text` | `var(--color-text-muted)` | 禁用态文字 |
| `--color-overlay` | `oklch(30% 0.05 240 / 0.4)` | 遮罩层（使用半透明的鸭蓝，而非半透明的纯黑） |
| `--color-focus-ring` | `oklch(76% 0.12 55 / 0.5)` | 焦点环（半透明暖珊瑚色） |

---

## CSS 使用示例

```css
/* 全局页面底色 */
body {
  background-color: var(--color-background);
  color: var(--color-text-primary); /* #1B3A47 深鸭蓝 */
}

/* 标志性的圆润按钮 */
.btn-primary {
  background: var(--color-primary); /* 暖珊瑚色 */
  color: var(--color-text-inverse);
}
.btn-primary:hover {
  background: var(--color-primary-hover);
}

/* 带有色彩倾向的遮罩层 */
.modal-overlay {
  background: var(--color-overlay); /* 半透明的深水鸭蓝 */
}
```
