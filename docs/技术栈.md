# 前端技术栈 (Frontend Tech Stack)

本项目是一个基于 Node.js 构建的**大型前端工程**，全面拥抱现代化前端最佳实践。我们在强调代码严谨性的同时，继续贯彻 Headspace 风格的疗愈与人文感。

## 核心框架与构建体系

- **核心 UI 库**: **React 18**
- **语言标准**: **TypeScript**。要求开启严格模式 (Strict Mode)，所有组件 Props 和状态必须强类型化，彻底消灭未定义的 `any` 类型。
- **构建工具**: **Vite**。提供极速的冷启动和 HMR (模块热替换)，替代老旧的构建系统。
- **包管理器**: 推荐使用 `npm`, `pnpm` 或 `yarn`。

## 样式与设计系统架构

在大工程体系下，我们将所有的设计系统落地为工程化配置：

- **原子化 CSS**: **Tailwind CSS (v3/v4)**。配合 PostCSS 集成到 Vite 构建流中，用于页面骨架搭建与基础排版。
- **设计 Tokens 的管理**:
  - 我们依然保留**原生 CSS Variables (`var(--color-...)`)** 的绝对控制权。
  - 所有之前在 `01-06` 文档中定义的颜色、大圆角 (Pill)、彩色弥散阴影等，将被统一抽离到一个全局的 `global.css` 或 `tokens.css` 中。
- **字体加载**: 本地引用或继续利用系统内置的 **LXGW WenKai (霞鹜文楷)** 字体，避免外部字体阻塞渲染。

## 动效物理引擎 (Framer Motion)

我们摒弃了纯 CSS 控制复杂动画的做法，全面引入 **Framer Motion** 来接管前端动画。

根据 `docs/07-motion-physics.md` 的设定，本项目追求的是**“绝对放松、慢节奏、无生硬弹簧”**的物理质感。因此在使用 Framer Motion 时，必须遵循以下特化规则：

1. **禁用高硬度 Spring**：默认不要使用高 stiffness 的弹簧效果。
2. **拥抱柔和贝塞尔**：优先使用 `type: "tween"`，并传入对应我们 CSS Tokens 的极缓贝塞尔曲线，如 `ease: [0.25, 1, 0.5, 1]` (Editorial Reveal)。
3. **时长控制 (Duration)**：Framer Motion 的默认时长偏短。请在 `transition` 中明确指定较长的 `duration`（例如微交互设为 `0.4`，入场设为 `0.8`，页面级翻页过渡设为 `1.2`）。
4. **共享形变 (Layout Morph)**：充分利用 `layoutId` 属性，实现如同液态流转、纸张展开的高级共享元素过渡。

## 开发与协作规范
1. 启动项目请运行 `npm run dev`。
2. 创建组件时，切勿在一个巨大的文件中堆砌代码。遵循“单一职责”的组件化拆分。
3. 如果修改了全局的 Design Tokens，请确保更新了对应的文档与全局 CSS 文件。
