# 间距系统

> **核心哲学**：Headspace 的排版强调极致的“呼吸感”与“留白”。我们废弃了诸如 4px, 12px 这样过于细密紧凑的阶梯，转而采用大跨度的间距尺度，营造从容不迫的阅读与交互体验。

---

## Spacing Scale (慷慨留白)

| Token | 值 | 质感描述 | 常见用途 |
|-------|-----|---------|----------|
| `--space-8` | `8px` | **微距 (Micro)** | 图标与文字的间距、极小卡片的内边距。 |
| `--space-16` | `16px` | **内聚 (Inner)** | 按钮的 Padding、紧凑列表项的间距。 |
| `--space-24` | `24px` | **基础 (Base)** | 卡片的标准 Padding、常规元素的间距。 |
| `--space-40` | `40px` | **舒朗 (Loose)** | 段落之间、核心区块内部不同组件的间距。 |
| `--space-64` | `64px` | **呼吸 (Breathe)** | Section 级别的小分区、页面的基础留白。 |
| `--space-96` | `96px` | **空灵 (Airy)** | 页面大模块间距、Hero 区块的上下留白。 |

> **强制规则**：除了极少数情况下用于微调的 `4px` 或 `2px`，所有的常规排版必须从上面的 6 个核心 Token 中选取。**不要**发明中间值（如 12, 20, 32），这些只会破坏整体的大气与呼吸感。

---

## Section Padding (容器内外边距)

确保页面内容不显得拥挤，在任何屏幕尺寸下都有充分的空气感。

| Token | 值 | 用途 |
|-------|-----|------|
| `--section-padding-y` | `clamp(40px, 5vw, 64px)` | 通用模块的上下留白 |
| `--section-padding-y-lg` | `clamp(64px, 8vw, 120px)` | Hero 区域、重点插画区域的巨大留白 |
| `--section-padding-x` | `clamp(24px, 4vw, 64px)` | 容器的两侧安全距离 |
| `--container-max` | `960px` | 主内容最大宽度（刻意收窄以确保行长利于阅读，不追求铺满屏幕） |
| `--container-narrow` | `640px` | 沉浸式阅读区域（纯文字文章段落）最大宽度 |

## 间距应用范例

### 圆润按钮 (Pill Button)
Headspace 的按钮通常非常肥硕，点击区域巨大。
```css
.btn {
  /* 垂直 16px，水平 40px 的极为宽泛的点击热区 */
  padding: var(--space-16) var(--space-40);
  gap: var(--space-8); /* 按钮内图标与文字的间距 */
}
```

### 疗愈卡片 (Wellness Card)
```css
.card {
  /* 内部大量留白 */
  padding: var(--space-40);
  /* 元素组之间的呼吸感 */
  gap: var(--space-24);
}
```

### 大段落排版 (Typography Spacing)
```css
.article-content > p {
  /* 段落之间拒绝紧凑，采用 40px 断行 */
  margin-bottom: var(--space-40);
}

h2.section-title {
  /* 标题下方留出 24px 空气 */
  margin-bottom: var(--space-24);
}
```

## Layout Gap 预设 (CSS Grid/Flex)

| Token | 值 | 用途 |
|-------|-----|------|
| `--gap-sm` | `var(--space-16)` | 卡片内的紧凑元素（如标题与副标题） |
| `--gap-md` | `var(--space-24)` | 并排的按钮组、插画与配文 |
| `--gap-lg` | `var(--space-40)` | 大网格卡片之间的缝隙 |
| `--gap-xl` | `var(--space-64)` | 核心内容板块的大跨度间隔 |
