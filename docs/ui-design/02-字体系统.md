# 字体系统

> 项目只使用 LXGW WenKai。文字要温暖、友好、松弛，但不引入额外手写字体或外部字体。

---

## Font Family

```css
:root {
  --font-heading: 'LXGW WenKai';
  --font-body: 'LXGW WenKai';
  --font-mono: 'LXGW WenKai Mono';
}
```

| 字体 | 来源 | 用途 |
| --- | --- | --- |
| LXGW WenKai | `@fontsource/lxgw-wenkai` 或本地自托管 | 标题、正文、UI 文字 |
| LXGW WenKai Mono | 本地自托管 | 代码块、终端、技术内容 |
| Caveat | Google Fonts | **仅限** `one-tree` 品牌 Logo 使用 |

禁止引入其他 Google Fonts（如 Quicksand、Nunito、Inter、Roboto 等）或 Arial、系统字体作为显示字体。情绪强调通过颜色、留白和排版节奏实现，不通过额外手写字体实现（品牌 Logo 除外）。

---

## Font Weight

| Token | 值 | 用途 |
| --- | --- | --- |
| `--font-weight-light` | `300` | 极淡提示、装饰性辅助文字 |
| `--font-weight-regular` | `400` | 正文、段落、普通控件 |
| `--font-weight-medium` | `500` | H1-H3、主按钮、强调文字 |

LXGW WenKai 仅使用 Light / Regular / Medium。禁止 Bold、SemiBold 和浏览器合成加粗。

## 排版规则

- 中文正文默认 `line-height: 2`，长文可用 `2.15`。
- 标题不使用负字距，`letter-spacing` 保持 `0`。
- 按钮文字使用 `line-height: 1`，高度由 padding 控制。
- 所有长段落使用 `text-wrap: pretty`。

```css
body {
  font-family: var(--font-body);
  font-size: var(--text-body);
  font-weight: var(--font-weight-regular);
  line-height: 2;
  color: var(--color-text-primary);
}

h1, h2, h3 {
  font-family: var(--font-heading);
  font-weight: var(--font-weight-medium);
  letter-spacing: 0;
  text-wrap: pretty;
}
```

## 字体加载

```css
@font-face {
  font-family: 'LXGW WenKai';
  src: url('/fonts/lxgw-wenkai-regular.woff2') format('woff2');
  font-weight: 400;
  font-display: swap;
}
```

只预加载首屏必需字重。其余字重按需加载，避免阻塞首屏。

## 避坑指南：字体无法生效的常见原因与修复方案

在开发过程中，常会遇到设置了 `LXGW WenKai` 或其他字体但界面依然渲染默认系统字体（如宋体/黑体）的问题。这通常是由以下几种情况导致，请在遇到排版问题时按此顺序排查：

1. **未在入口文件中引入网络字体**
   - **表现**：代码中使用了 `fontFamily: 'Caveat'` 或其他特定字体，但浏览器降级。
   - **修复**：如果是如 `Caveat` 这类特定用于局部品牌字体的 web font，必须确保在全局 `index.html` 的 `<head>` 标签中正确注入了 Google Fonts 链接，否则组件层面写死名字也无法下载资源。

2. **滥用硬编码而非系统 CSS 变量 (Token)**
   - **表现**：直接在组件里写 `style={{ fontFamily: 'Playfair Display, Noto Serif SC, serif' }}`。
   - **修复**：**永远不要硬编码英文字体名**。必须使用 `var(--font-heading)` 或 `var(--font-body)` 等经过全局定义的 CSS Variables。React 中 inline-style 的正确写法为 `style={{ fontFamily: 'var(--font-heading)' }}`。

3. **CSS 变量名 (Token Name) 拼写错误**
   - **表现**：写了变量 `font-family: var(--font-family-serif, serif);` 但依然回退到 `serif`。
   - **修复**：检查 `tokens.css`。系统仅暴漏了 `--font-heading`、`--font-body` 和 `--font-mono` 三个变量，不存在 `--font-family-serif`。错误拼写会导致变量失效并触发兜底值。
