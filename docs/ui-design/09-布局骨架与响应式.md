# 布局骨架与响应式

> 开发顺序：先搭页面框架，再放内容。布局骨架必须先于 Hero、组件细节和动效。

---

## 页面骨架

每个页面至少包含：

- `AppShell`：全局主题、背景、路由出口。
- `Header`：顶部导航，移动端要有明确的收起方案。
- `Main`：页面主内容，必须有可跳转语义入口。
- `Footer`：内容少时粘底，内容多时自然排在末尾。

```tsx
export interface AppShellProps {
  children: React.ReactNode;
}

export function AppShell({ children }: AppShellProps) {
  return (
    <div className="app-shell">
      <Header />
      <main className="page-main">{children}</main>
      <Footer />
    </div>
  );
}
```

## 容器系统

| Token | 值 | 用途 |
| --- | --- | --- |
| `--container-narrow` | `640px` | 长文、表单、阅读流 |
| `--container-default` | `960px` | 常规页面主内容 |
| `--container-wide` | `1120px` | 多列卡片、工作台 |
| `--container-full` | `1280px` | 极少数仪表盘或对比视图 |

布局默认不追求铺满屏幕。Headspace meditation 风格需要足够留白，优先保证阅读节奏。

```css
.container {
  inline-size: min(100% - var(--section-padding-x) * 2, var(--container-default));
  margin-inline: auto;
}
```

---

## 断点

| 名称 | 宽度 | 目标 |
| --- | --- | --- |
| Mobile | `375px` 起验收 | 单列、底部留白、触控优先 |
| Tablet | `768px` 起验收 | 双列可选，导航仍需克制 |
| Desktop | `1280px` 起验收 | 多列、宽容器、完整导航 |

Playwright 验收必须覆盖 `375px`、`768px`、`1280px`。

## Grid

```css
.content-grid {
  display: grid;
  grid-template-columns: repeat(12, minmax(0, 1fr));
  gap: var(--gap-lg);
}

@media (max-width: 767px) {
  .content-grid {
    grid-template-columns: 1fr;
    gap: var(--gap-md);
  }
}
```

移动端不要强行保留桌面列结构。图片、卡片、侧栏都应按阅读顺序自然堆叠。

---

## Header

- 桌面端可以水平导航，但项目入口数量要克制。
- 移动端使用全屏或底部抽屉式菜单，避免拥挤的横向滚动。
- Header 背景可用 `--glass-bg`，但必须保持文字对比度。
- fixed/sticky Header 必须给 Main 留出安全距离，不能遮挡首屏标题。

---

## Footer 粘底

```css
.app-shell {
  min-block-size: 100svh;
  display: flex;
  flex-direction: column;
}

.page-main {
  flex: 1;
}
```

---

## 首屏规则

- Hero 不一定要满屏；必须在常见桌面视口露出下一段内容的线索。
- 移动端首屏不能被装饰光晕、导航或按钮挤到需要横向滚动。
- 图片和插画必须服务内容，不用随机装饰物填空。
