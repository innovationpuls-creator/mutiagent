# 圆角与阴影

> 几何语言是柔软、圆润、可触摸。阴影是彩色环境光，不是冷灰投影。

---

## Border Radius Scale

| Token | 值 | 用途 |
| --- | --- | --- |
| `--radius-sm` | `16px` | 输入框、小标签、内层控件 |
| `--radius-md` | `24px` | 常规卡片、列表项 |
| `--radius-lg` | `32px` | 大卡片、模态框、Hero 面板 |
| `--radius-xl` | `40px` | 特殊沉浸式容器 |
| `--radius-full` | `9999px` | 按钮、Badge、头像、状态 pill |

规则：

- 页面中不出现小于 `--radius-sm` 的圆角。
- 所有按钮和主要操作入口使用 `--radius-full`。
- 大型内容块优先使用 `--radius-lg` 或 `--radius-xl`。

---

## Shadow Scale

所有阴影必须多层叠加。第一层负责贴地感，第二层负责弥散，第三层只在高层级中使用。

| Token | 值 | 用途 |
| --- | --- | --- |
| `--shadow-sm` | `0 1px 2px oklch(50% 0.05 55 / 0.08), 0 8px 24px oklch(76% 0.12 55 / 0.10)` | 输入框、静止卡片 |
| `--shadow-md` | `0 2px 6px oklch(50% 0.05 55 / 0.08), 0 18px 48px oklch(76% 0.12 55 / 0.16)` | hover 卡片、浮层 |
| `--shadow-lg` | `0 4px 12px oklch(50% 0.05 55 / 0.10), 0 32px 80px oklch(76% 0.12 55 / 0.22), 0 64px 120px oklch(84% 0.12 63 / 0.14)` | 模态框、Hero 焦点 |
| `--shadow-inset` | `inset 0 1px 2px oklch(20% 0.04 235 / 0.18), inset 0 -1px 1px oklch(100% 0 0 / 0.18)` | 凹槽、进度条轨道 |
| `--shadow-glow` | `0 0 48px oklch(76% 0.12 55 / 0.24), 0 0 96px oklch(84% 0.12 63 / 0.16)` | 极少数品牌高光 |

## 交互层级

```css
.wellness-card {
  border-radius: var(--radius-lg);
  background: var(--color-surface);
  box-shadow: var(--shadow-sm);
  transition:
    transform var(--duration-lazy-hover) var(--ease-lazy),
    opacity var(--duration-lazy-hover) var(--ease-lazy);
}

.wellness-card:hover {
  transform: translateY(calc(var(--space-4) * -1)) scale(1.01);
  box-shadow: var(--shadow-md);
}
```

---

## 禁止事项

- 禁止单层阴影。
- 禁止纯黑、冷灰投影。
- 禁止硬边卡片。
- 禁止通过边框左色条制造廉价强调。
- 禁止为了密度牺牲圆角与留白。
